# Архитектура и описание работы программы

## Обзор

**Трекер питания (ver3cm)** — однофайловое браузерное приложение без серверной части (SPA — Single Page Application). Все вычисления выполняются в браузере пользователя. Приложение не требует установки и работает с любого веб-сервера (включая GitHub Pages).

## Принцип работы

### Навигация

Все разделы (Дневник, Дашборд, Профиль, Упражнения, Справка) присутствуют в DOM одновременно как `<section>` элементы. При переходе по вкладкам JavaScript просто переключает `display: none / block`.

```js
// Пример: показать раздел
function pokazatRazdel(nazvanie) {
    // скрыть все, показать нужный
}
```

### Управление состоянием

Все данные хранятся в единственном глобальном объекте `sostoyanie`:

```js
var sostoyanie = {
    tekushayaData: '2026-03-25', // выбранная дата
    profil: { pol, vozrast, rost, ves, ... },
    dnevnik: { '2026-03-25': { zavtrak: [...], obed: [...], ... } },
    uprazhneniya: { '2026-03-25': [...] },
    yazyk: 'ru'
};
```

### Интернационализация (I18N)

Все тексты интерфейса хранятся в объекте `PEREVODY`:
```js
var PEREVODY = { ru: { appTitle: 'Трекер питания', ... }, en: { ... } };
```

HTML-элементы помечаются атрибутом `data-i18n="ключ"`. При переключении языка функция `primenytYazyk()` обходит DOM и заменяет тексты.

### Загрузка данных

При старте загружаются три JSON-файла через `fetch()`:
- `data/foods.json` → `baza_produktov`
- `data/vitamins.json` → `vitaminyDannyje`
- `data/minerals.json` → `mineralyDannyje`

### Пользователи

Список пользователей определён в `user.js` (массив `SPISOK_POLZOVATELEY`). При выборе пользователя его JSON-файл загружается через `fetch()` и данные мерджатся в `sostoyanie`.

Сохранение — через `Blob` + ссылка для скачивания:
```js
var blob = new Blob([JSON.stringify(dannye)], { type: 'application/json' });
var a = document.createElement('a');
a.href = URL.createObjectURL(blob);
a.download = 'user_data.json';
a.click();
```

## Разделы script.js

| Раздел | Содержимое |
|--------|------------|
| 1. Состояние | Объект `sostoyanie` |
| 2. Переводы | Объект `PEREVODY` (ru/en) |
| 3. Данные | Загрузка foods.json, vitamins.json, minerals.json |
| 4. Навигация | `pokazatRazdel()` |
| 5. Дневник | Дата, приёмы пищи, расчёт итого |
| 6. Поиск продуктов | Фильтрация и добавление |
| 7. Профиль | ИМТ, BMR, TDEE, цели |
| 8. Дашборд | Карточки, недельный график, прогресс нутриентов |
| 9. Упражнения | Добавление и удаление |
| 10. Пользователи | Загрузка, сохранение, скачивание |
| 11. Язык | `pereklyuchitYazyk()`, `primenytYazyk()` |
| 12. Справка | Загрузка Markdown через fetch |
| 13. Инициализация | `init()`, обработчики событий |

## Добавление нового пользователя

1. Откройте `user.js` и добавьте запись:
```js
{ id: 'NewUser', nazvanie: 'Новый пользователь', fajl: 'User/NewUser.json' }
```
2. Создайте файл `User/NewUser.json` по образцу `User/Test1.json`

## Расширение базы продуктов

Откройте `data/foods.json` и добавьте новый объект в массив `products`. Структуру см. в `data/data_info.md`.
