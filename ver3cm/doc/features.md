## Проблемы при копировании cronometer.com и предложения по их решению

### 1. Ограниченная база продуктов
**Проблема:** cronometer.com содержит сотни тысяч продуктов. В нашей базе `foods.json` — около 40 продуктов.

**Предложение:**
- Подключить USDA FoodData Central API для поиска в реальном времени
- Или скачать и конвертировать открытый датасет (например, Open Food Facts)
- Добавить возможность ручного добавления продуктов пользователем

### 2. Работа без backend (файловая система)
**Проблема:** В браузере нельзя записывать файлы напрямую. Данные теряются при перезагрузке.

**Предложение:**
- Использовать `localStorage` для автосохранения данных между сессиями
- При открытии страницы проверять наличие сохранённых данных и предлагать их загрузить

### 3. Ошибка «Failed to fetch» при открытии с файловой системы (file://)
**Проблема:** `fetch()` не работает на протоколе `file://` в некоторых браузерах (Chrome).

**Решение:**
- Использовать GitHub Pages или локальный сервер (`python3 -m http.server`)
- Список пользователей вынесен в `user.js` (не JSON) именно для обхода этой проблемы

### 4. Отсутствие базы рецептов
**Проблема:** cronometer.com позволяет создавать рецепты из нескольких ингредиентов.

**Предложение:** Добавить раздел «Рецепты» — коллекция из нескольких продуктов с общим КБЖУ.

### 5. Нет учёта воды
**Проблема:** cronometer.com отслеживает потребление воды.

**Предложение:** Добавить счётчик воды (стаканы/мл) в дневник или на дашборд.

### 6. Нет синхронизации между устройствами
**Проблема:** Данные хранятся только в памяти браузера текущего сеанса.

**Предложение:** Добавить экспорт/импорт через QR-код или интеграцию с Google Drive.
